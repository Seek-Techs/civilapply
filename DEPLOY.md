# CivilApply — Deployment Guide (Render.com)

## What you need before starting
- A **GitHub account** (free) — https://github.com
- A **Render account** (free) — https://render.com
- Your project folder ready on your computer

---

## Step 1 — Push your code to GitHub

Open a terminal in your project folder (`civil_engineering/`) and run:

```bash
git init
git add .
git commit -m "Initial deploy"
```

Then go to https://github.com/new, create a **private** repository called `civilapply`, and follow the instructions to push:

```bash
git remote add origin https://github.com/YOUR_USERNAME/civilapply.git
git branch -M main
git push -u origin main
```

**Important:** Make sure `.env` is in your `.gitignore` (it already is) — never push API keys to GitHub.

---

## Step 2 — Deploy on Render

1. Go to https://render.com and sign up / log in
2. Click **"New +"** → **"Web Service"**
3. Connect your GitHub account and select the `civilapply` repository
4. Render will auto-detect the `render.yaml` file — click **"Apply"**

Render will:
- Install dependencies from `requirements.txt`
- Start the app with Gunicorn (production server)
- Create a 1GB persistent disk at `/data` for your SQLite database

---

## Step 3 — Set your environment variables

In the Render dashboard → your service → **"Environment"** tab, add:

| Key | Value | Required? |
|-----|-------|-----------|
| `FLASK_SECRET` | (auto-generated by Render) | Yes |
| `SMTP_EMAIL` | your Gmail address | For email sending |
| `SMTP_PASSWORD` | your Gmail App Password | For email sending |
| `GROQ_API_KEY` | your Groq key | For AI CV tailoring |
| `COHERE_API_KEY` | your Cohere key | Alternative to Groq |

To get a Gmail App Password:
1. Go to https://myaccount.google.com/apppasswords
2. Select "Mail" and your device
3. Copy the 16-character password

---

## Step 4 — Your live URL

After deploy (takes ~3 minutes), Render gives you a URL like:
```
https://civilapply.onrender.com
```

That's your live app — share it with anyone.

---

## Free tier limits (Render)

| What | Limit |
|------|-------|
| Monthly hours | 750 hrs (enough for 1 service) |
| Sleep after inactivity | 15 minutes of no traffic |
| Disk | 1 GB (plenty for SQLite + CVs) |
| Custom domain | Free with your own domain |

**Note on sleep:** On the free tier, the app "sleeps" after 15 mins of no traffic. The first request after sleeping takes ~30 seconds to wake up. To avoid this, upgrade to the $7/month Starter plan.

## Adding a custom domain (optional)

1. Buy a domain (e.g. `civilapply.ng`) from Namecheap (~$10/year)
2. In Render → your service → **"Custom Domains"**
3. Add your domain and follow the DNS instructions

## Troubleshooting

**App crashes on start:**
Check Render logs → "Logs" tab. Most common cause: missing environment variable.

**"Module not found" error:**
A dependency is missing from `requirements.txt`. Add it and push again.

**Database resets on redeploy:**
Make sure the disk is mounted at `/data` in `render.yaml` — the app already saves `applications.db` there.

**Email not sending:**
Double-check `SMTP_EMAIL` and `SMTP_PASSWORD` in Render env vars. Use a Gmail App Password, not your regular Gmail password.
